# Subject: 2k_to_do_list 

## Descripción del proyecto
Crea una aplicación web interactiva de lista de tareas utilizando **React**. La aplicación debe permitir al usuario gestionar sus tareas diarias: agregar, marcar como completadas y eliminar tareas. El proyecto tiene como objetivo enseñar fundamentos de React, manejo del estado, componentes reutilizables y buenas prácticas en desarrollo frontend.

* Conceptos: Componentes, Props, State, Eventos.
* Aprendizaje: Crear, eliminar y marcar tareas completadas.

## Objetivos de aprendizaje
Al completar este proyecto, deberías ser capaz de:

### React Básico y Avanzado
- Crear componentes funcionales y de clase.  
- Gestionar estado con `useState` y/o `useReducer`.  
- Pasar datos entre componentes usando props.  

### Interactividad y Eventos
- Capturar eventos de usuario (clicks, formularios).  
- Actualizar la interfaz de manera reactiva.  

### Buenas prácticas en Frontend
- Organizar componentes de manera modular.  
- Escribir código limpio y mantenible.  
- Aplicar estilos consistentes usando CSS o frameworks como Tailwind.  

### Persistencia básica (opcional)
- Guardar tareas en `localStorage` para mantenerlas entre sesiones.  

### Despliegue y documentación
- Publicar tu proyecto en GitHub Pages, Vercel o Netlify.  
- Crear un README claro y profesional describiendo la aplicación y su uso.  

## Requisitos funcionales
- Crear una interfaz que muestre la lista de tareas. 
- Permitir al usuario agregar nuevas tareas con un formulario.  
- Permitir marcar tareas como completadas.  
- Permitir eliminar tareas individuales.  
- Actualizar la interfaz automáticamente al modificar las tareas.  
- (Opcional) Guardar las tareas en `localStorage` para persistencia.  

## Desafíos extra
- Implementar filtrado de tareas: todas, completadas o pendientes.  
- Agregar prioridad o categorías a las tareas.  
- Añadir animaciones al agregar o eliminar tareas.  
- Usar `useContext` o Redux para manejar estado global si la app crece.  

## Entregables
- Código fuente completo en un repositorio de GitHub.  
- Capturas de pantalla o GIF mostrando la aplicación en funcionamiento.
